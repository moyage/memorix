# Contributing to Memorix

Thank you for your interest in contributing to Memorix!

## Development Philosophy

Memorix follows a zero-dependency philosophy:
- Only `@modelcontextprotocol/server` and `better-sqlite3` are allowed as runtime dependencies
- No heavy frameworks
- Minimal configuration with sensible defaults

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes following the existing code style
4. Run tests if applicable
5. Commit your changes with a descriptive message
6. Push to your fork and submit a pull request

## Coding Standards

- Use CommonJS (Node.js native) - no ES modules
- Follow existing naming conventions
- Keep functions focused and single-purpose
- Add comments for complex logic

## Reporting Issues

- Use GitHub Issues for bug reports
- Include reproduction steps when possible
- Specify your environment (Node.js version, OS, etc.)
